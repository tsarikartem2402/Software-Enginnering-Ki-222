# Лабораторна робота №4: Робота із версійністю баз даних

Цей проект демонструє використання **Liquibase** для керування версіями бази даних на прикладі бази даних **H2**.

## 🛠️ Виконані кроки

### 1. Встановлення та налаштування
- Встановлено **OpenJDK 17** та **Liquibase 4.27.0**.
- Налаштовано файл `liquibase.properties` для підключення до локальної бази даних H2.
- Створено початковий файл змін `changelog.xml`.

### 2. Створення схеми бази даних
- Додано `changeSet` з `id="1"` для створення таблиці `lab4_test` з полями `id` (Primary Key) та `name`.
- Виконано команду для застосування змін:
  ```bash
  ./liquibase update
  ```

### 3. Версіонування та модифікація
- Додано тег `version1` для фіксації поточного стану бази даних (`changeSet id="2-tag"`).
- Додано нову колонку `description` до таблиці `lab4_test` (`changeSet id="3-add-column"`).
- Знову виконано оновлення для застосування нових змін.

### 4. Відкат змін (Rollback)
- Продемонстровано можливість повернення до попередньої версії бази даних за допомогою команди:
  ```bash
  ./liquibase rollback --tag=version1
  ```
- Після виконання команди колонка `description` була видалена автоматично, що підтверджено перевіркою схеми таблиці.

## 📁 Структура проекту
- `liquibase.properties` — конфігурація підключення до БД.
- `changelog.xml` — файл з історією змін (changesets).
- `README.md` — опис лабораторної роботи.

## 🚀 Як запустити проект
1. Переконайтеся, що у вас встановлена Java.
2. Завантажте Liquibase.
3. Виконайте команду для перевірки статусу:
   ```bash
   liquibase status
   ```
4. Застосуйте зміни:
   ```bash
   liquibase update
   ```

## 📊 Liquibase Hub (Крок 10)
Для інтеграції з Liquibase Hub необхідно:
1. Зареєструватися на [hub.liquibase.com](https://hub.liquibase.com).
2. Отримати `API Key`.
3. Виконати команду:
   ```bash
   liquibase register-changelog
   ```
Це дозволить переглядати історію операцій `update` та `rollback` через графічний веб-інтерфейс.
