# MongoApp - Лабораторна робота №3

Цей проєкт є частиною лабораторної роботи №3 з дисципліни "Інженерія програмного забезпечення". Він демонструє роботу Node.js (Express) додатка з базою даних MongoDB через Docker Compose, а також автоматизоване тестування продуктивності за допомогою k6.

## Структура проєкту

- `app.js`: Основний файл сервера Express.
- `public/`: Статичні файли (HTML/JS для фронтенду).
- `Dockerfile`: Інструкції для збірки образу додатка.
- `docker-compose.yaml`: Конфігурація для одночасного запуску додатка та MongoDB.
- `k6/`: Сценарії для навантажувального тестування.

## Локальний запуск

Для запуску всього стеку (Node.js + MongoDB) виконайте команду:

```bash
docker-compose up --build
```

Додаток буде доступний за адресою `http://localhost:3000`.

## Тестування продуктивності (k6)

У папці `k6/` містяться три типи тестів:
1. `load_test.js` — стандартний тест навантаження.
2. `stress_test.js` — стрес-тест із поступовим нарощуванням користувачів.
3. `ci_thresholds_test.js` — тест із визначеними порогами (Thresholds) для CI/CD.

## CI/CD Pipeline

Проєкт інтегровано з GitHub Actions (`.github/workflows/mongo-app-ci.yml`). При кожному пуші в папку `lab3/mongoapp/`:
- Автоматично розгортається середовище в Docker.
- Виконуються всі сценарії тестування k6.
- Перевіряються встановлені пороги продуктивності.
